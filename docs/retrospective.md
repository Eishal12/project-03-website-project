# Project retrospective- phtotography portfolio

## what went well
- The project scope was clear from the beginning, which made planning easier.
- Setting up the repository and organizing the docs folder went smoothly.
- Building the two page website was simple because the design was minimal and focused.
- Using a small set of photos helped keep the layout clean and manageable.

## what went wrong
- some tasks took longer than expected, especially deciding which photos to include.
- A few technical issues came up with linking pages and styling which required extra testing.
- It was easy to get distracted trying to make the design good.

# what i would do differently
- start the website layout earlier to leave more time for testing.
- Choose the photos before writing the scope to avoid changing ideas later.
- Keep the design even simpler to avoid overthinking small details.
