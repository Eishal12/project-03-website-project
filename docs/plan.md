# project plan- photography portfolio 

## Tame risk analysis
- **Time limits** - mitigate: work small steps and commit often.
- **Technical issues** - mitigate: test links, layout, and github pages early.
- **Not enough photos** - Accept: use a small but strong selection
- **over-designing** - Tolerate: keep layout minimal and simple.
- **Adding extra features** - eliminate: Follow the scope and aviod new ideas.

## work breakdown structure
- **planning:** finalize scope, choose photos, outline pages.
- **setup:** create repo, add README, create doc folder, add scope + plan.
- **Development:** build index.html, about html, style.css, add categories
- **Testing:** check navigation, layout, and responsiveness.
- **finalization:** write retrospective, publis with github pages, review repo.
